Public engineering documentation project at [100daydash.blog](https://100daydash.blog/) for showing how platform, automation, and dashboard work develops over time.

100DayDash documents independent engineering work across cloud infrastructure, CI/CD, DevSecOps, observability, API-driven data workflows, and dashboard development. The writing shows design decisions, implementation tradeoffs, operational lessons, and repeatable delivery patterns rather than only finished screenshots.

Current focus areas:

- Public documentation of platform engineering decisions and tradeoffs
- Cloud automation and Infrastructure as Code project notes
- DevSecOps workflow documentation for repository governance and CI/CD
- Dashboard engineering with Power BI, Looker Studio, Hugo, Astro, and Python-based visualization
- Data platform development using APIs, PostgreSQL/Supabase patterns, and modeled analytical outputs
